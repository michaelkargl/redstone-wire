## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Create all resource files needed for the three new block types: blockstates, models, item models, and language entries.

Blockstate files (in assets/redstone_wire/blockstates/):
- redstone_input.json — 16 power variants (power=0 → redstone_input model, power=1-15 → redstone_input_powered model)
- redstone_output.json — 16 power variants (power=0 → redstone_output model, power=1-15 → redstone_output_powered model)
- redstone_connector.json — Single variant (no power property, just default model)

Block models (in assets/redstone_wire/models/block/):
- redstone_input.json — cube_all, initially reuse redstone_chain texture
- redstone_input_powered.json — cube_all, initially reuse redstone_chain_powered texture
- redstone_output.json — cube_all, initially reuse redstone_chain texture
- redstone_output_powered.json — cube_all, initially reuse redstone_chain_powered texture
- redstone_connector.json — cube_all, initially reuse redstone_chain texture

Item models (in assets/redstone_wire/models/item/):
- redstone_input.json — parent: redstone_wire:block/redstone_input
- redstone_output.json — parent: redstone_wire:block/redstone_output
- redstone_connector.json — parent: redstone_wire:block/redstone_connector

Language file updates (en_us.json):
- "block.redstone_wire.redstone_input": "Redstone Input"
- "block.redstone_wire.redstone_output": "Redstone Output"
- "block.redstone_wire.redstone_connector": "Redstone Connector"

Note: Custom textures can be added later. Initially reuse existing textures so the blocks are functional.
